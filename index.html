Теория · Блок 1
Основы n8n
Ключевые концепции которые нужно понять до первого воркфлоу. Нажимай на карточки — они раскрываются.
Что такое n8n
n8n — инструмент автоматизации с открытым кодом. Соединяешь сервисы между собой через визуальную схему из блоков (нодов) — без написания backend-кода. Можно запускать в облаке (n8n.cloud) или на своём сервере.
Отличие от Zapier и Make: в n8n есть Code нод на JavaScript — это даёт гибкость для сложных сценариев, которые no-code инструменты не потянут.
⚡
Воркфлоу ▼
Цепочка нодов
Воркфлоу — схема автоматизации. Состоит из нодов, соединённых стрелками. Данные текут слева направо.

Active — работает автоматически по триггеру
Inactive — только ручной запуск

Можно экспортировать как JSON и импортировать в другой инстанс — удобно для шаблонов.
🔲
Нод ▼
Один блок = одно действие
Нод делает одну вещь: отправить запрос, записать строку, проверить условие. Получает items на входе (Input), обрабатывает и передаёт дальше (Output).

Trigger — первый нод, запускает воркфлоу
Action — действие с внешним сервисом
Core — логика: IF, Switch, Code, Set

Переименовывай ноды — это важно для expressions и читаемости.
📦
Item ▼
Единица данных
Item — один объект данных в потоке. Внутри каждого item есть объект json с полями.

Пример: Webhook получил 3 заявки → это 3 items. Каждый нод ниже обработает каждый item по очереди автоматически.

Структура: { json: { name: "Анна", email: "a@b.com" } }

Поле binary — для файлов (PDF, картинки). В большинстве задач работаем только с json.
🚀
Триггер ▼
Точка запуска
Триггер — всегда первый нод. Ждёт событие и запускает всё остальное. Без активного триггера воркфлоу не запустится автоматически.

Webhook — входящий HTTP-запрос
Schedule — по расписанию / cron
Telegram Trigger — сообщение боту
Manual — только вручную (для теста)

Один воркфлоу = один триггер.
🔑
Credentials ▼
Доступы к сервисам
Credentials — сохранённые токены, API-ключи, OAuth-данные. Создаются один раз в Settings → Credentials и переиспользуются в любом ноде. Хранятся зашифрованно.

API Key — Telegram, OpenAI, большинство сервисов
OAuth2 — Google (Sheets, Gmail, Drive)
Basic Auth — логин/пароль для старых API

Давай им понятные имена: "Telegram Bot @mybot", "Google Sheets prod".
🔄
Execution ▼
Один запуск воркфлоу
Execution — один запуск воркфлоу от начала до конца. Каждый запуск сохраняется в разделе "Executions" — там видно какие данные прошли через каждый нод и где была ошибка.

Success — зелёный, всё прошло
Error — красный, нод упал
Waiting — воркфлоу ждёт ответа

Executions — главный инструмент отладки. Всегда смотри туда если что-то пошло не так.
Как тестировать воркфлоу
В n8n два режима: тестовый (кнопка "Test Workflow" или "Execute Node") и продакшн (воркфлоу активен и работает по триггеру).

В тестовом режиме данные видны прямо в интерфейсе у каждого нода — нажимаешь на нод и смотришь Input / Output. Это самый удобный способ понять что происходит внутри.
Совет: тестируй каждый нод по отдельности кнопкой "Execute Node" прежде чем запускать весь воркфлоу целиком.

Ключевые ноды
Ноды которые используются в 90% реальных автоматизаций. Для каждого — настройки, данные на выходе, типичные ошибки.

Триггеры

Telegram Trigger
▼
Слушает входящие сообщения Telegram-бота. Запускает воркфлоу каждый раз когда пользователь пишет боту.
ПолеЧто ставить
CredentialTelegram API credentials с токеном бота
Updatesmessage — для текстовых сообщений. callback_query — для кнопок.
Данные из триггера
$json.message.text — текст сообщения
$json.message.chat.id — Chat ID (куда отвечать)
$json.message.from.first_name — имя пользователя
$json.message.from.id — ID пользователя
Воркфлоу с Telegram Trigger должен быть активирован (toggle вверху). В тестовом режиме ловит только первое сообщение после нажатия "Listen for events".
Webhook
▼
Принимает HTTP-запросы (GET, POST) на уникальный URL. Используется для форм, сайтов, интеграций.
ПолеЧто ставить
HTTP MethodPOST — для форм и данных. GET — для простых запросов.
PathПуть URL, например lead-form. Итоговый URL: /webhook/lead-form
Response ModeLast Node — если используешь "Respond to Webhook" нод
Данные из вебхука
POST тело: $json.body.fieldName
Query params: $json.query.paramName
Headers: $json.headers['content-type']
Два URL: Test URL (работает пока слушаешь в редакторе) и Production URL (когда воркфлоу активен). Используй Test URL при разработке.
Schedule Trigger
▼
Запускает воркфлоу по расписанию. Поддерживает cron и визуальный настройщик.
CronКогда запускается
0 9 * * 1-5Каждый будний день в 9:00
0 17 * * 5Каждую пятницу в 17:00
0 */2 * * *Каждые 2 часа
30 9 * * 1,3,5Пн, Ср, Пт в 9:30
Schedule Trigger не передаёт никаких данных — он просто запускает цепочку. Первые данные появляются в следующем ноде.

Действия

Telegram — Send Message
▼
Отправляет сообщение через Telegram-бота. Поддерживает Markdown, HTML, кнопки.
ПолеЧто ставить
Chat IDID чата. Для ответа пользователю: {{ $('Telegram Trigger').item.json.message.chat.id }}
TextТекст сообщения. Поддерживает {{ expressions }}
Parse ModeMarkdown или HTML для форматирования
Markdown в сообщениях
*жирный* → жирный   _курсив_ → курсив   `код` → моноширинный
[текст](url) → ссылка
Google Sheets
▼
Читает, добавляет и обновляет строки. Требует OAuth2 credentials.
ОперацияКогда использовать
Append RowДобавить новую строку в конец таблицы
Get Many RowsПрочитать строки с фильтром или все сразу
Update RowОбновить конкретную строку по номеру или значению
Для Append Row включи режим "Map Each Column Manually" — так сам указываешь какое поле идёт в какую колонку. Надёжнее автоматического маппинга.
AI Agent
▼
Отправляет запрос в языковую модель (OpenAI и др.) и возвращает текстовый ответ.
ПолеЧто ставить
ModelGPT-4o, GPT-3.5-turbo и др.
System MessageИнструкция для AI — роль, правила, контекст. Статичный текст.
User MessageСообщение пользователя. Обычно expression из предыдущего нода.
CredentialOpenAI API Key
Ответ AI-агента
Ответ модели доступен как $json.output в следующем ноде.
Из нода через несколько шагов: $('AI Agent').item.json.output
Если System Message пустой — AI будет отвечать непредсказуемо. Всегда задавай роль и ограничения.

Логика

IF
▼
Делит поток на две ветки: true (условие выполнено) и false (не выполнено).
Тип условияПример
Строка содержит$json.email contains @
Не пустое$json.name is not empty
Число больше$json.count > 0
Равно значению$json.status equals approved
Можно добавить несколько условий через AND/OR — например проверить что и имя и email заполнены одновременно.
Code
▼
Выполняет JavaScript. Для трансформации данных, работы с массивом или сложной логики.
Собрать все items в строку
const items = $input.all(); const summary = items.map((item, idx) => `${idx + 1}. ${item.json.name} — ${item.json.request}` ).join('\n'); return [{ json: { summary, count: items.length } }];
Сгенерировать уникальный ID
const id = `plan_${Date.now()}_${Math.random().toString(36).substr(2,6)}`; return [{ json: { ...$input.first().json, id } }];
Переключай режим нода: "Run Once for All Items" (все items сразу) или "Run Once for Each Item" (по одному).
Set
▼
Добавляет, изменяет или удаляет поля в item. Самый частый утилитарный нод.
РежимЧто делает
Add FieldДобавляет новое поле, остальные сохраняются
Keep Only SetУдаляет все поля кроме указанных
Set нод удобен чтобы переименовать поле: получил user_name, хочешь передать дальше как name — Set сделает без кода.
Теория · Блок 3
Expressions
Как динамически подставлять данные из предыдущих нодов в поля. Без этого воркфлоу будут статичными.
Что такое expression
Expression — JavaScript-выражение в двойных фигурных скобках: {{ ... }}. Вставляется в любое текстовое поле нода. Вычисляется в момент выполнения и подставляет реальное значение.

Вместо статичного "Привет!" пишешь Привет, {{ $json.name }}! — каждый пользователь получит своё имя.

Основные переменные

ПеременнаяЧто возвращает
$json.fieldNameПоле из текущего item (данные последнего нода)
$json.body.fieldПоле из тела Webhook POST-запроса
$('Имя нода').item.json.fieldДанные из конкретного нода по его имени
$input.all()Все items из предыдущего нода как массив
$input.first().jsonПервый item из предыдущего нода
$nowТекущее время (объект DateTime)
$now.toISO()Время в ISO: 2024-11-15T09:00:00
$now.toFormat('dd.MM.yyyy')Дата: 15.11.2024
$now.toFormat('dd.MM.yyyy HH:mm')Дата + время: 15.11.2024 09:30

Примеры

В поле Text у Telegram нода
Привет, {{ $('Telegram Trigger').item.json.message.from.first_name }}! *Новая заявка* Имя: {{ $('Webhook').item.json.body.name }} Email: {{ $('Webhook').item.json.body.email }} Дата: {{ $now.toFormat('dd.MM.yyyy HH:mm') }}
В поле User Message у AI Agent
// Передать текст из Telegram {{ $('Telegram Trigger').item.json.message.text }} // Передать данные из формы с контекстом Клиент {{ $json.body.name }} пишет: {{ $json.body.request }}
Chat ID — куда отвечать
{{ $('Telegram Trigger').item.json.message.chat.id }}
После AI Agent нода $json содержит ответ AI, а не данные из Telegram. Поэтому Chat ID берём через $('Telegram Trigger').item.json, а не через $json.
